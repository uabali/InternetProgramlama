ADIM 1: GIT KONFIGÜRASYONU (İlk Kez Yapılacak)
----------------------------------------------

# Git kullanıcı adınızı ayarlayın
git config --global user.name "uabali"

# Git email adresinizi ayarlayın (GitHub hesabınızdaki email)
git config --global user.email "your-email@example.com"

# Kontrol edin
git config --global user.name
git config --global user.email


ADIM 2: GITHUB'DA YENİ REPOSITORY OLUŞTURUN
-------------------------------------------

1. https://github.com adresine gidin
2. Sağ üstten "+" işaretine tıklayın
3. "New repository" seçin
4. Repository adı: InternetProgramlama (veya istediğiniz ad)
5. Public veya Private seçin
6. "Create repository" tıklayın
7. Repository URL'sini kopyalayın (örnek: https://github.com/uabali/InternetProgramlama.git)


ADIM 3A: EĞER KLASÖR ZATEN GIT REPOSITORY İSE
---------------------------------------------

cd /home/abali/InternetProgramlama

# Değişiklikleri ekleyin
git add .

# Commit yapın
git commit -m "Homework_4: Kütüphane Yönetim Sistemi eklendi"

# Remote ekleyin (eğer yoksa)
git remote add origin https://github.com/uabali/InternetProgramlama.git

# Veya mevcut remote'u değiştirin
git remote set-url origin https://github.com/uabali/InternetProgramlama.git

# Push yapın
git push -u origin main

# Eğer "main" yerine "master" branch'i varsa:
git push -u origin master


ADIM 3B: EĞER KLASÖR GIT REPOSITORY DEĞİLSE
-------------------------------------------

cd /home/abali/InternetProgramlama

# Git repository başlat
git init

# Tüm dosyaları ekle

OLASI HATALAR VE ÇÖZÜMLER
=========================

HATA 1: "remote origin already exists"
ÇÖZÜM:
git remote remove origin
git remote add origin https://github.com/uabali/REPO_ADINIZ.git

HATA 2: "failed to push some refs"
ÇÖZÜM:
git pull origin main --rebase
git push origin main

HATA 3: "Permission denied (publickey)"
ÇÖZÜM:
- HTTPS kullanın: https://github.com/uabali/REPO_ADINIZ.git
- Veya SSH key oluşturun

HATA 4: "username/password" soruyor
ÇÖZÜM:
- Personal Access Token (PAT) kullanın
- GitHub Settings > Developer Settings > Personal Access Tokens


PERSONAL ACCESS TOKEN (PAT) OLUŞTURMA
=====================================

1. GitHub'da Settings > Developer Settings > Personal Access Tokens
2. "Generate new token (classic)" tıklayın
3. Token'a bir isim verin
4. "repo" yetkisini seçin
5. Token'ı oluşturun ve KOPYALAYıN (bir daha göremezsiniz!)
6. Push yaparken username yerine GitHub kullanıcı adınız
7. Password yerine token'ı yapıştırın


GIT KOMUTLARI ÖZET
==================

git init                    # Repository başlat
git status                  # Durum kontrolü
git add .                   # Tüm dosyaları ekle
git add dosya.txt           # Tek dosya ekle
git commit -m "mesaj"       # Commit yap
git push                    # GitHub'a gönder
git pull                    # GitHub'dan çek
git log                     # Commit geçmişi
git branch                  # Branch listesi
git remote -v               # Remote bilgisi
